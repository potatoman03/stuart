## Design standards

### For cheat sheets / reference cards
- Use **multi-column layout** via reportlab Table with invisible borders (2-3 columns)
- Dense, scannable content — no wasted space
- Color-coded sections with consistent heading styles
- Use callout boxes for key formulas, exam tips, common mistakes
- Professional color palette: define 4-5 colors as constants (e.g., primary blue `#2962FF`, accent, section headers, body text, muted)

### For study guides / reports
- Clean single-column layout with proper heading hierarchy
- Page headers/footers with page numbers
- Table of contents for longer documents
- Section dividers between major topics

### General
- Define all colors, fonts, and spacing as constants at the top of the script
- Use `getSampleStyleSheet()` as base, then customize with `ParagraphStyle`
- **CRITICAL:** `getSampleStyleSheet()` already defines styles named "Bullet", "Definition", "Code", "Title", "Heading1", etc. When adding custom styles, use unique names like "MyBullet", "SectionHead", "BodyText2", etc. to avoid `KeyError: "Style already defined"`.
- For charts: use matplotlib with a professional style (`plt.style.use('seaborn-v0_8-whitegrid')` or similar), save to `BytesIO`, embed as `Image`
- Tables: zebra striping, colored headers, proper padding
- Footer on every page: "Generated by Stuart"
